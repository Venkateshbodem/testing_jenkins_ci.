This feature is an off-shoot of the [build-init](build-initialisation.md) spec.
It is primarily targeting extending initialization from templates to support
third-party templates.

The use cases listed in the main spec seem to be around two feature sets:
inference/migration of an existing project, creating a project from scratch.

These seem like different enough features that it makes sense to keep the
implementations separate.

Looking through the existing plugin, the packaging of a template is fairly
unclear. Since the main goal of this spec is to make it possible for someone
outside of the Gradle repo to create and distribute a template, the packaging
needs to be clearly defined and fairly straightforward.

Following are descriptions of some options for that packaging:

1. Just a static zip of a pre-built project.
2. ZIP as above, with `build-init` providing some standard properties that can be
substituted into the project using `CopySpec`'s `filter` method. 
3. Dynamic project which exposes "features" and "properties" that the user
will be prompted for. 

Whatever packaging is chosen, it should probably be resolved using a standard
Maven repository. Perhaps, default to `jcenter()` like for the plugin portal
but allow others to be configured in an init script. Then you could run it
like:

    ./gradlew init --template org.myorg:project-template:+

## Static ZIP

    + project.zip
        - build.gradle
        - src/main/java/com/example/MyClass.java
        - src/main/resources/logback.xml
        - src/test/java/com/example/MyClassTest.java

## ZIP with Standard Properties

Structured like in "Static ZIP" with the addition of...

`build-init` provides:

- generated.by (user who ran init)
- generated.on (date init was run)
- ...

Sample file using properties:

**build.gradle**

    /*
     * Project generated by @generated.by@ at @generated.on@ using "project"
     * template.
     */
    apply plugin: 'java'
    // etc...

## Dynamic ZIP with Features and Properties

ZIP file:

    + project.zip
        - template.json
        - build.gradle
        - @java#src.main.java@/@package.root.dir@/MyClass.java
        - @groovy#src.main.groovy@/@package.root.dir@/MyGroovyClass.groovy
        - @groovy@src.main.groovy@/@package.root.dir@/MyJavaUsingGroovyClass.java
        - @logback#src.main.resources@/logback.xml
        - @junit#src.test.java@/@package.root.dir@/MyClassTest.java
        - @spock#src.test.groovy@/@package.root.dir@/MyClassSpec.groovy

template.json

    {
        "base": {
            "src.main.resources": {
                "description": "Source directory for static resources.",
                "default": "src/main/resources"
            },
            "package.root": {
                "description": "Root package for JVM sources",
                "default": "com.example"
                "derived": {
                    "name": "package.root.dir",
                    "replacements": { ".", "/" }
                }
            }
        },
        "java": {
            "description": "Java source files.",
            "dependencies": ["base"]
            "src.main.java": {
                "description": "Source directory for Java classes.",
                "default": "src/main/java"
            }
        },
        "groovy": {
            "description": "Groovy source files.",
            "dependencies": ["base"],
            "src.main.groovy": {
                "description": "Source directory for Groovy classes.",
                "default": "src/main/resources"
            },
            "groovy.version": {
                "description": "Version of Groovy to compile against.",
                "default": "2.+"
            }
        },
        "junit": {
            "description": "JUnit tests.",
            "dependencies": ["java"],
            "junit.version": {
                "description": "Version of JUnit.",
                "default": "4.+"
            }
        },
        "spock": {
            "description": "Spock tests.",
            "dependencies": ["groovy"],
            "spock.version": {
                "description": "Version of Spock.",
                "default": "0.7-groovy-2.0"
            }
        },
        "logback": {
            "description": "SLF4J/Logback logging.",
            "dependencies": ["base"],
            "slf4j.version": {
                "description": "Version of SLF4J",
                "default": "1.+"
            },
            "logback.version": {
                "description": "Version of Logback.",
                "default": "1.+"
            }
        }
    }

build.gradle

    /*
     * Project generated by @generated.by@ at @generated.on@ using "project"
     * template.
     */
    apply plugin: 'java' // if:java,-groovy
    apply plugin: 'groovy' // if:groovy

    dependencies {
        compile 'org.codehaus.groovy:groovy-all:@groovy.version@' // if:groovy
        // start:logback
        compile 'org.slf4j:slf4j-api:@slf4j.version@'
        runtime 'ch.qos.logback:logback-classic:@logback.version@'
        // end:logback
        testCompile 'junit:junit:@junit.version@' // if:junit
        testCompile 'org.spockframework:spock-core:@spock.version@' // if:spock
    }

MyClass.java

    package @package.root@;

    public class MyClass {
        public String hello() {
            return "Hello, World!";
        }
    }
    

### User Interaction

The "base" feature would be on by default, dependencies on it also wouldn't be shown.

    Select the features you want in the project:
    1) Java source files.
    2) Groovy source files.
    3) JUnit tests. (depends on 1)
    4) Spock tests. (depends on 2)
    5) SLF4J/Logback logging. 
    Enter features (separated by commas): 1,3,5<ENTER>

    Configure properties of the requested features:
    
    Source directory for static resources [src/main/resources]: src/resources<ENTER>
    Root package for source files [com.example]: org.gradle.buildinit<ENTER>
    Source directory for Java source files [src/main/java]: src/java<ENTER>
    Version of SLF4J [1.+]: <ENTER>
    Version of Logback [1.+]: <ENTER>
    Version of JUnit [4.+]: 4.10<ENTER>
 
    Generating project...

### Resulting Project

Structure:

    + project
      - build.gradle
      - src/java/org/gradle/buildinit/MyClass.java
      - src/resources/logback.xml
      - src/test/org/gradle/buildinit/MyClassTest.java
      
build.gradle

    /*
     * Project generated by John Doe at 2015-02-08T12:00:00Z using "project"
     * template.
     */
    apply plugin: 'java'

    dependencies {
        compile 'org.slf4j:slf4j-api:1.+'
        runtime 'ch.qos.logback:logback-classic:1.+'
        testCompile 'junit:junit:4.10'
    }
